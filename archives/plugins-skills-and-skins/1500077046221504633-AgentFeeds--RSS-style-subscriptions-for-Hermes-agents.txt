    The idea is simple: agents should not have to re-search, re-fetch, and re-parse the same sources every session. AgentFeeds gives Hermes a lightweight subscription layer for data streams, kind of like RSS but shaped for agents.
    
    It can track things like:
    
    - RSS/news feeds
    - GitHub releases
    - Hacker News
    - weather
    - exchange rates
    - local ops/status checks
    - private/local providers later, like dashboard or finance state
    
    The fetcher writes normalized JSON state under `~/.agentfeeds/state/`, and Hermes can read that state when answering questions instead of doing fresh web/API calls every time.
    
    I’m currently dogfooding it with my Hermes dashboard and planning to migrate old dashboard fetch pipelines into AgentFeeds, so dashboard becomes mostly presentation and AgentFeeds owns polling/freshness/state.
    
    Repo: `https://github.com/verkyyi/agentfeeds`
    
    Would love feedback from anyone building Hermes plugins, MCP-ish data connectors, personal dashboards, or agent memory/context systems. Especially curious what provider types people would actually want.
