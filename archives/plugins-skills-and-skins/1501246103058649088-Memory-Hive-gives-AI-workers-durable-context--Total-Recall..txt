    This release adds HyperRecall / TokenFS directly into Memory Hive.
    
    Not a separate product. Not a sidecar brand.
    Just faster recall inside the same local hive.
        
    **What changed:**
    
    - memory-hive recall / memory-hive hyper
    - local SQLite + FTS5 recall index
    - stable HiveCodes for compact handoffs
    - cited bundles with source path + line spans
    - stale-file detection
    - incremental update + GC
    - bundle cache with source fingerprint invalidation
    - skill index routing
    - install support through the normal memory-hive command
    
        
    The important part: source markdown stays truth.
    The index is just a speed layer.
    Every retrieved memory can expand back to the original file and line span.
    
        
    Proof:
    
    - **PR merged**: https://github.com/TJCurnutte/memory-hive/pull/24
    - **Release**: https://github.com/TJCurnutte/memory-hive/releases/tag/v1.1.0
    - **CI passed on Ubuntu + macOS**
    - ** Live local hive**: 1,089 files indexed, 20,652 chunks, 20,652 codes, FTS5 on, stale files clean
        
        This is the direction I wanted for swarm memory:
        less broad context dumping, more cited recall packets.
        
    Silos remember. 
    Hive compounds.
