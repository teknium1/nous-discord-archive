    Hey everyone! Excited to share another Hermes plugin that I shipped.
    
    Meet PlanForge, a spec-driven development planning plugin that keeps your Hermes agent from going off the rails on complex projects. You know the problem: you give your agent a big task, and it starts coding before thinking, skips steps, or loses track of where it is halfway through. PlanForge fixes that.
    
    How it works:
    
    You get a disciplined 5-phase workflow: init -> plan -> execute -> verify -> ship
    
    - Phase locking prevents the agent from jumping ahead or working on multiple phases at once
    - Wave-based execution groups related tasks so the agent can parallelize safely
    - Auto-verification runs your test suite, linter, and build checks after each phase
    - Persistent state survives session restarts (uses Mnemosyne under the hood)
    - GitHub integration creates branches, commits, and PRs automatically
    
    Quick start:
    ```
    # In your Hermes config
    plugins:
      enabled:
        - planforge
    ```
    ```
    # Then in your project
    /planforge-init      # Creates .planning/ with PROJECT.md, REQUIREMENTS.md, ROADMAP.md
    /planforge-plan      # Plans the current phase
    /planforge-execute   # Runs wave-based execution
    /planforge-verify    # Checks against acceptance criteria
    /planforge-ship      # Branch, commit, push, PR
    ```
    
    It generates actual spec files (PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md) so you always know what the agent is doing and why. Phase progress, blockers, and decisions are tracked and persisted.
    
    Repo: https://github.com/AxDSan/planforge-hermes
    
    Would love feedback, feature requests, or contributions. What planning workflows are you all using with Hermes right now?
