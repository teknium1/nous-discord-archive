    Built a skill that connects Hermes to Ethereum Attestation Service on Base mainnet. The agent autonomously:
    
    1. Registers its own identity onchain (schema: owner, agentName, version, platform)
    2. Creates a task completion attestation with keccak256 content hash
    3. Links task to identity via refUID - verifiable chain of trust
    4. Verifies both attestations and saves UIDs to agent memory
    
    One prompt, three onchain transactions, full verification. No manual script execution - Hermes reads the SKILL.md, runs the commands, and reports back with explorer links.
    
    **Why this matters**
    IETF is drafting agent capability attestation extensions. AIS-1 just launched on Base. OASIS published an Agentic IAM framework in April 2026. Agent identity is the next infrastructure layer - this skill is a starting point.
    
    **Install**
    
    `git clone https://github.com/Zhekinmaksim/hermes-attest-skill ~/.hermes/skills/attest
    cd ~/.hermes/skills/attest && npm install
    echo "ATTEST_PRIVATE_KEY=0x..." > .env
    echo "ATTEST_NETWORK=base" >> .env
    echo "ATTEST_RPC_URL=https://mainnet.base.org" >> .env`
    
    Then just ask Hermes:
    
    `Register my agent identity on Base and create a task attestation for the skill release.`
    
    Works on Sepolia (free), Ethereum, Optimism, Base, Arbitrum.
