    It is a standalone Weather MCP server powered by rustwx, so agents can work with real model/satellite/radar data directly instead of relying on a fixed web dashboard. Some of the features are quickly being expanded to CONUS and worldwide later (specifically GOES-18 sat plots) and some features like the meteograms are much quicker if you tell your agent a specific location you're interested in so it can build a custom new data store made for temporal data (HRRR only for now). 
    
    https://github.com/FahrenheitResearch/hermes-weather-agent
    
    What it can do:
    
    - Render HRRR, GFS, ECMWF Open Data, and RRFS-A weather maps
    - Resolve the latest available model run automatically
    - Render direct model fields like temperature, winds, heights, RH, QPF, reflectivity, and pressure
    - Render derived/severe products like CAPE, SRH, shear, STP, ECAPE, UH tracks, lapse rates, and QPF windows
    - Generate GOES satellite imagery, including GeoColor, GLM overlay, RGB products, and ABI bands
    - Render NEXRAD Level-II radar products, including base products and dual-pol products
    - Generate point meteograms for arbitrary lat/lon points
    - Warm local meteogram stores for repeated point/time-series queries
    - Render Skew-T soundings
    - Render HRRR VolumeStore cross sections for arbitrary routes
    - Build small local HRRR data packs so repeated queries do not re-download the same data
    - Use HRRR `.idx` byte-range fetching where possible to avoid pulling full GRIB files
    - Report cache size, disk usage, and top storage consumers
    - Run background jobs for heavier products and long workflows
    - Build showcase HTML galleries of generated products
    - Expose everything through MCP so Hermes, Claude, or any MCP client can call it
    
    Example workflows:
    
    - “Show me the latest HRRR temperature, wind, and RH maps for southern California.”
    - “Make a meteogram for this incident location for the next 18 hours.”
    - “Draw a cross section from Redding to Reno and render theta-e, RH, wind, and fire-weather products.”
    - “Pull the latest GOES-West GeoColor, fire RGB, and GLM overlay for California.”
    - “Render KTLX radar base reflectivity, velocity, and dual-pol products.”
    - “Compare latest HRRR CAPE, SRH, shear, and STP over the southern plains.”
    - “Build a small local HRRR data pack for the latest few hours so my agent can keep working offline-ish.”
    - “Tell me how much disk the weather cache is using and what can be evicted.”

    I think all the weather nerd in here are gonna like it. I spent a lot of time removing all dependencies possible including matplotlib/contourf. You can essentially run your own pivotalweather style website with this via hermes-agent. Im confident that theres nothing else in the world that can plot faster, process faster, and compute faster. Every calc has been verified as a direct equivalent to MetPy, and ECAPE products are a full ecape-parcel-py full parcel entraining CAPE written totally in rust which allows you to make ECAPE product maps. If you were to try to make the same maps with ecape-parcel-py, it would take 120 hours per map roughly (7-8s per point calc), and you can now do it fully controlled by hermes-agent in 30 seconds.
