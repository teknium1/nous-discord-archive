     # DMN — Default Mode Network memory for Hermes Agent
    
    Instead of storing and retrieving facts like a trad database, the DMN runs a bg thread that continuously wanders memories, generates reflections, and consolidates the index in the background — the same way the brain's default mode network processes experience during rest. 
    
    **What makes it different:**
    - **No vectors, no embeddings, no GPU.** Pure BM25 inverted index — fast, local, deterministic retrieval with no external dependencies
    - **Self-organising.** Entity relationships, preferences, and recurring themes emerge from the statistical pressure of language. No extraction pipelines, no fragile graph schemas — the index is the graph
    - **Self-pruning.** Memories constitute as they're processed, without the aid of agentic llm calls. The index never grows unbounded, never needs manual curation
    - **Orphan handling.** When a memory drifts loose from the network with no BM25 neighbours, it gets handed to an autonomous spike agent (generates a cron task) with its full toolset to decide what to do with it
    - **Mood-aware.** Arousal level (0–100) derived from index pressure drives thought generation style, system prompt tone, and prefetch behaviour across named states 
    - **Per-profile customisable.** All LLM-facing prompts, mood vocabulary, and spike briefings live in a `DREAM.md` file you can override per Hermes profile — no code changes
    
    Drop it in `plugins/memory/dmn/`, run `hermes memory setup`, done.
