    https://github.com/briancaffey/hermes-otel
    This plugin sends traces, metrics and logs to one or more OpenTelemetry collectors. It collects both session and cron job data. I have tested a bunch of different OTel collectors: Phoenix, Langfuse, LangSmith, SigNoz, Jaeger, Tempo/LGTM. The repo includes docker compose references for each backend. There are a few different PRs in the hermes-agent github repo that add OpenTelemetry, but I think this can be handled by an plugin that listens to hooks. 
    
    One limitation is that the pre/post api request hooks do not include the input ouput for intermediary LLM requests. You also don't have a way to view the system prompt since it is not provided in the hooks. You *can* see the input/output from the post_llm_call hook. 
    
    I'm curious to know how others are doing observability for hermes agent. 
    
    Here a GitHub pages documentation site for the hermes-otel plugin: https://briancaffey.github.io/hermes-otel/
