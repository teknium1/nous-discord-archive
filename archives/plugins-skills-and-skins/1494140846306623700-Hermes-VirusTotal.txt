    Hermes VirusTotal is a Hermes Agent plugin that adds VirusTotal  (https://www.virustotal.com) reputation signals for file and hash workflows. It provides reputation context that the model and the user can use when deciding what to trust.
    
    https://github.com/king-tero/hermes-virustotal
    
    Installation:
    
    **hermes plugins install king-tero/hermes-virustotal**
    
    Restart the gateway if you use Hermes through a messaging platform:  hermes gateway restart
    
    Features:
    
    Reputation tools:
    * vt_check_hash(hash): check a MD5, SHA1, or SHA256 in VTAI/VirusTotal.
    * vt_check_file(path): hash a file in the active Hermes execution environment and check that hash.
    
    Advisor context: 
    observed artifacts are summarized via pre_llm_call as fenced, untrusted context for the model. Context is scoped to the active Hermes session, deduplicated between turns, and aged out when stale.
    
    Provenance metadata: 
    content created by write_file, patch, and execute_code is hashed and recorded with its VTAI/VirusTotal verdict when available.
    
    Privacy-respecting upload policy: 
    binaries (ELF, PE/MZ, Mach-O, WASM, Java class, DEX) are auto-submitted to VTAI by default so the community can analyze potential new malware.  Scripts, source code, markdown, and other text are never auto-uploaded. Archives are opt-in. A path blocklist and a secret-pattern scan short-circuit uploads that could leak user data.
    
    Optional enforcement: 
    set VTAI_ENFORCE_KNOWN_MALICIOUS=1 to block only exact content hashes that VTAI/VirusTotal reports as malicious. This is off by default.
