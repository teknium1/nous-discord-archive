    Been building H-OPS, an operator dashboard for Hermes Kanban.
    The goal is to make multi-agent work observable.
    You can:
    
    • create and assign tasks
    • route work to agent profiles
    • track ready/running/blocked/done lanes
    • inspect progress, output, and logs
    • see stale workers
    • create parent/child task dependencies
    • view connected work as a DAG-style workflow graph
    • monitor each agent profile, model, env, skills, and last seen state
    
    The workflow graph is the part I really wanted.
    Kanban is great for status.
    But once agent work starts branching, you also need structure.
    What depends on what?
    Which task is blocking the next one?
    What unlocks after this finishes?
    H-OPS lets you see that chain instead of guessing from a flat list of tickets.
    It’s starting to feel less like a todo board, and more like a control room for autonomous agents.
