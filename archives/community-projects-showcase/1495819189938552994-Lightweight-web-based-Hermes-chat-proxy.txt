    ---
    🔴 Important (Security/Reliability)
    1. Fixed PBKDF2 Salt ⚠️  
    The salt is hardcoded (b'hermes-proxy-signing-key'). Same password = same key across all deployments.  
    Fix: Generate random salt at first startup, store alongside .env.
    2. No Per-User Session Isolation ⚠️  
    Single-user by design, but authenticated users can access any session ID in the DB.  
    Fix: Scope session lookups to a user identifier if going multi-user.
    3. Unbounded Memory Growth ⚠️  
    browser_sessions dict grows forever (no TTL/LRU). Long-running servers may OOM.  
    Fix: Add functools.lru_cache or TTL eviction.
    4. Broad Exception Handling ⚠️  
    except Exception masks real errors as "Invalid JSON".  
    Fix: Catch specific exceptions (json.JSONDecodeError, ValidationError).
    5. CSRF on Logout (Low risk)  
    POST /auth/logout has no CSRF token. Mitigated by SameSite=Strict, but worth adding.
    ---
    🟡 Nice-to-Have (Enhancements)
    6. SQLite WAL Mode  
    PRAGMA journal_mode=WAL for better concurrent reads (if Hermes enables it).
    7. ETag for Static Files  
    Improve caching with Starlette's ETag middleware.
    8. Granular HTTP Timeouts  
    Use httpx.Timeout(connect=10, read=300) instead of single 300s timeout.
    9. Native EventSource  
    Replace manual SSE parsing with EventSource API for cleaner reconnection logic.
    10. HTTPS Enforcement  
    Fail fast if secure=True cookie flag is set but running HTTP (dev mode detection).
    ---
    ✅ What's Done Well
    - ✅ HMAC-signed stateless cookies (PBKDF2, 100k iterations)
    - ✅ Rate limiting (5 attempts / 60s)
    - ✅ Security headers (CSP, X-Frame-Options, etc.)
    - ✅ SRI hashes on CDN deps
    - ✅ DOMPurify XSS protection
    - ✅ Session-lost detection UX
