    https://github.com/alias8818/hermes-tool-slimmer
    "Hermes Tool Slimmer reduces repeated tool-schema overhead by selecting the smallest useful tool set for a turn. It builds an indexable corpus from Hermes tool schemas, ranks candidate tools with local BM25 plus explicit boosts, and fails open to the original schema list when anything goes wrong."

    ### Other strategies
    
    (This is just stuff AI suggested, and I don't know offhand whether it's correct.)
    
    - Adjust the compression threshold from the default 0.5 to a more aggressive setting like 0.3 for high-volume messaging.
    - Reduce `protect_last_n` to lessen long-term history accumulation.
    - Use the `hermes tools` command to disable unused tools.
    - Exclude irrelevant tools (apparently even stripping `browser_ tools` for messaging platforms can save over 1,200 tokens per request)
