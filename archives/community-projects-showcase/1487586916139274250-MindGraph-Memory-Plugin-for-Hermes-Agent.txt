[2026-03-28T22:59:18.836000+00:00] shan.rizvi (id=1487586916139274250)
    Built a plugin that gives Hermes agents persistent semantic graph memory across sessions, using the new lifecycle hooks from v0.5.0.
    
    **What it does:**
    When you install this plugin, your agent automatically gets:
    - Session context at conversation start — active goals, open decisions, governance policies, weak claims from your knowledge graph
    - Per-turn semantic retrieval — score-gated, so it only injects context when it's actually relevant to what you're talking about
    - All injected ephemerally (no prompt cache breakage)
    
    **Install:**
    pip install hermes-mindgraph-plugin
    or
    hermes plugins install shuruheel/hermes-mindgraph-plugin
    
    Then set MINDGRAPH_API_KEY in ~/.hermes/.env and you're done. The plugin is auto-discovered, no config needed. You can get it for free at https://mindgraph.cloud/
    
    **How it works under the hood:**
    Three hooks — on_session_start (opens a MindGraph session, pre-fetches goals/decisions), pre_llm_call (returns context dict for ephemeral injection), on_session_end (closes session). Every API call is wrapped in try/except so a MindGraph failure never breaks your conversation.
    
    This is also a reference for anyone wanting to build memory plugins on the new hook system — the whole thing is ~140 lines.
    
    GitHub: https://github.com/shuruheel/hermes-mindgraph-plugin
    PyPI: https://pypi.org/project/hermes-mindgraph-plugin/
    MindGraph: https://mindgraph.cloud
