    Bug Fix PR #7358: 
    
    Problem:When two messages hit the gateway at the same time, `os.environ` values like `HERMES_SESSION_THREAD_ID` get overwritten by each other. Result: notifications go to the wrong thread because `os.environ` is process-global and shared across all async tasks.
    
    Solution: Replaced `os.environ` with Python's `contextvars.ContextVar`. Each async task now gets its own isolated copy — no more cross-task contamination. Added `gateway/session_context.py` for context var definitions and updated all tools that were reading from `os.environ`.
    
    PR: https://github.com/NousResearch/hermes-agent/pull/7391
