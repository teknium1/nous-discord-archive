    This is a good idea if done right. Because popularity of AI agents is still a bleeding-edge topic, the people writing documentation and wikis are the same people that use AI  most heavily. Unfortunately, IME this results in documentation being incomprehensible AI-slop. Official OpenClaw documentation is very bad at this. Official Hermes docs are very bad, but not as bad as OpenClaw's, and the few wiki articles that I browsed from your site are not really any better. What is needed for all of these things is a human editor, and it doesn't seem that anyone bothers to edit the slop before posting it. Or if I'm being charitable, perhaps they do edit it, but they are not good at expressing clear thoughts in writing, so the edited product is still not useful.
    
    I love OSS and have contributed code to projects in the past. I like the idea of your wiki. I want to edit articles myself and make pull requests to help improve it, but it's impossible to do without understanding what the articles are trying to say in the first place.
    
    Example of the problem from the Hermes official docs: https://hermes-agent.nousresearch.com/docs/integrations/providers#named-custom-providers
    I am trying to figure out how to configure custom providers and have them accessible via their TUI chat interface `/model` command.
    The docs never explain the difference between the top-level keys `providers:` and `custom_providers:` in `config.yaml`.  In fact, that page doesn't even mention the `providers:` key. For that, you have to go to another page entirely: https://hermes-agent.nousresearch.com/docs/user-guide/configuration
    
    So their docs are contradictory and there is no comprehensive `config.yaml` reference anywhere. No explanation about how subkeys keys in the yaml affect the logic of model resolution. But there are plenty of bullet lists and markdown charts with circular references to other topics.
