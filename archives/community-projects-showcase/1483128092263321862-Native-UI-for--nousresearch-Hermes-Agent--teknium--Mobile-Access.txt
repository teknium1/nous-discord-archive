[2026-03-18T04:40:59.457000+00:00] outsourc_e (id=1483686635454468230)
    **Connecting Hermes Workspace to your Hermes Agent** 🔧
    > 
    > Quick answer: the Workspace needs our **WebAPI fork** — the upstream `hermes-agent` doesn't include the FastAPI server yet.
    > 
    > **Setup (2 min):**
    > ```bash
    > # 1. Backend — use our fork (has the WebAPI)
    > git clone https://github.com/outsourc-e/hermes-agent.git
    > cd hermes-agent
    > python -m venv .venv && source .venv/bin/activate
    > pip install -e .
    > hermes setup          # configure your API key
    > hermes webapi         # starts FastAPI on :8642
    > 
    > # 2. Frontend — in a new terminal
    > git clone https://github.com/outsourc-e/hermes-workspace.git
    > cd hermes-workspace
    > pnpm install && pnpm dev   # opens on :3000
    > ```
    > 
    > **Already installed upstream?** Just pull our fork instead:
    > ```bash
    > cd hermes-agent
    > git remote add webapi https://github.com/outsourc-e/hermes-agent.git
    > git fetch webapi && git merge webapi/main
    > pip install -e . && hermes webapi
    > ```
    > 
    > **Latest workspace update** (just pushed): auto-detects which gateway APIs are available. If endpoints are missing, it shows you exactly what's unsupported instead of crashing. Check your terminal for:
    > ```
    > [gateway] available: health, models; missing: sessions, skills, memory
    > ```
    > 
    > The upstream `hermes gateway` command is a process manager for Telegram/Discord bridges — it's different from `hermes webapi` which runs the REST API the workspace connects to.
