    Finally cleaned this up for public release. hermes-lab is the bookkeeper for running experiments autonomously. you give it a search space and a way to evaluate results, it handles scheduling, tracking, and suggesting what to try next. inspired by the autoresearch wave (karpathy, sakana ai scientist, AIDE) but generic -- works for ml training, hyperparameter sweeps, code mutation, whatever.
    
    how i use it: hermes-agent runs experiments all day in the background. i check in from telegram to see progress, what improved, and queue up draft ideas. come back later to a summary of everything tried.
    
    4 execution modes:
    - cron -- autonomous background runs on a schedule
    - burst -- rapid back-to-back iterations
    - guided -- pauses for approval each step
    - swarm -- multiple strategies rotate sharing a blackboard
    
    other highlights:
    - file-first, no database or server
    - 6 search strategies: random, perturb, bayesian, evolution, tree search (AIDE-inspired), llm
    - multi-fidelity: cheap proxy runs first, promote winners to expensive validation
    - dispatch protocol for remote workers/cloud GPUs
    - agent-agnostic: best with hermes-agent, works with any agent harness
    
    working example you can run in 2 minutes with zero api keys - clone and try labctl burst.
    
    https://github.com/amenti-labs/hermes-lab
    
    happy to answer questions.
