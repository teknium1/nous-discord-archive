    Hermes-Relay v0.4.0 is live on Github — Google Play in review! 🚀
    
    Native Android app for Hermes — built because I wanted a direct, no-middleman experience with the Hermes system and interface.
    
    Chat — streaming, session management, slash commands, tool visualization, file attachments, message queuing.
    Voice — real-time voice conversation with TTS/STT, morphing sphere visualizer, and voice-to-bridge intent routing.
    Bridge — the agent can read your screen, tap, type, swipe, long-press, drag, control media, access clipboard, send texts, make calls, and more — behind a five-stage safety gate system.

    v0.4.0 Released — Bridge Feature Expansion + Google Play! 🚀
    
    Big one. v0.4 roughly triples the bridge surface from v0.3. The agent can now long-press, drag, read/write clipboard, control media playback, fire raw Android Intents, search the accessibility tree with filters, detect screen changes without re-downloading the whole tree, run batched macro workflows, and more — all still behind the same five-stage safety gate system.
    
    Sideload-only goodies:
    - Direct SMS, one-tap calling, contact lookup, and GPS location — so voice commands like "text Mom saying on my way" or "call Hannah" just work
    - Voice → bridge intent routing with destructive-verb confirmation (the agent asks before it sends)
    - Vision-driven android_navigate for close-the-loop UI automation
    
    Google Play version is in review! 🥳 I'll drop the open testing link here as soon as it's live. The Play build is a conservative feature set (chat, voice, notifications — no agent device control) to stay within Play's Accessibility policy. Sideload remains the full-capability build for those who want the agent to actually do things on your phone.
    
    Other highlights:
    - Per-app skill playbooks so the agent knows common flows without fumbling
    - Wake-lock-guarded gesture dispatch — commands land even on dim/idle screens
    - Accessibility event stream — agent can wait for UI events instead of polling
    - Multi-window screen reads (notification shade, popups, system UI)
    - Tons of bug fixes from on-device testing
    
    GitHub: https://github.com/Codename-11/hermes-relay/releases/tag/v0.4.0
    Docs: https://codename-11.github.io/hermes-relay/
    
    APK sideload is available now on the release page. Would love feedback — especially on voice-to-bridge intents and the new phone control tools if anyone's brave enough to let their agent send texts for them 😎
