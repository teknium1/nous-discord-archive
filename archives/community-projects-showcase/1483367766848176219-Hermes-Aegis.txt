    Security hardening layer for Hermes Agent — Uses a transparent MITM proxy that prevents secret leakage, dangerous command execution, unauthorized data exfiltration and more.  
    
    Originally built for the hackathon, but I will be continuing to use Hermes so I'll be keeping it updated and working on fixes as they pop up for me. Would also love to see anything the team thinks is useful integrated into the main hermes-agent project 🙂
    
    This project aims to address the one thing really holding me back from using Hermes - the security consideration. While it might be fine to run it on a proper hosted setup somewhere, I just wanted to keep using my personal laptop as its the most powerful device I have right now, I'm comfy on it, all my stuff is here etc etc. Buuuut all my stuff being on there is also a concern. And I can't even really think about getting this onto my work device - even though I'm Hermes-pilling my dev team day by day. So I asked myself what it would take to make someone fully comfortable to install Hermes on a work device. This project is at least a step in that direction, although I will be the first to admit that this probably still has gaps that are missed. Any feedback is appreciated!
    
    https://github.com/Tranquil-Flow/hermes-aegis
    https://x.com/FlowTranquil/status/2032897035779842425
    
    --*Security Features*--
    **Blocks secret exfiltration** — scans all outbound HTTP for API keys, tokens, credentials
    **Injects API keys** — secrets stay in your encrypted vault, never in agent memory
    **Detects dangerous commands** — 35 risky patterns (shell injection, destructive ops, privesc, SSH exfiltration)
    **Blocks dangerous commands in gateway mode** — Patch 5 enforces blocking when AEGIS_ACTIVE=1
    **Blocks SSH/non-HTTP exfiltration** — Docker network isolation + SSH/SCP/netcat command detection
    **Reactive audit agents** — automated investigation and circuit breaker response to security events
    **Scheduled reports** — periodic audit digest generation via hermes cron system
    **Graceful session resume **— shows session ID and resume command on exit
    **Rate-limits bursts** — detects suspicious data tunneling patterns; escalates to blocking on repeated anomalies
    **Restricts domains **— optional allowlist for outbound connections
    **Audit trail **— tamper-proof hash-chained log of all security events
    **Tirith content scanning **— proxy-level inspection of LLM responses for homograph URLs, code injection, terminal escapes
    **Approval system** — pluggable backends (block/log_only/webhook) with persistent pattern cache and TTL rules
    **Container isolation **— Docker mode with AEGIS_CONTAINER_ISOLATED=1 and container handshake protocol

    --*Notes for anyone trying this*--
    For me right now it works great, but this is after debugging a bit. I've tried to ensure that wherever possible all existing integrations should work, but there is a lot that is untested. For running with backend:docker, and just needing the Discord integration you're fine - outside of that... uncharted territory.
    
    If you want to easily be able to fall back to using the normal hermes-agent, don't delete your credentials file when prompted in the install process (this is normally fine to do as this happens after they have been encrypted into your vault, and you can unencrypt them afterwards as needed).
    
    If you do try this out and get any issues just @ me! Ask Hermes for an incident report, or try provide as much info as possible so I can effectively debug and I'll try get it fixed as soon as possible.

    **Hermes-Aegis v0.2.0 is out** 
    
    https://github.com/Tranquil-Flow/hermes-aegis/releases/tag/v0.2.0
    
    This release rolls up everything added since **v0.1.9** and brings Aegis up to date with the latest Hermes Agent line.
    
    **Headline changes**
    - **Hermes v0.11 compatibility:** updated patch targets and runtime glue for the current Hermes provider routing, Docker, command approval, gateway, and sandbox behavior.
    - **Simpler launch:** `hermes-aegis` now starts protected Hermes directly. `hermes-aegis run` still works, but is no longer required.
    - **One-command updates:** `hermes update` now calls `hermes-aegis update`, so Hermes and Aegis can update together and Aegis patches are re-applied automatically.
    - **Hybrid plugin package:** added `plugins/hermes-aegis/` with plugin metadata, hook entrypoints, state helpers, dashboard API scaffolding, and plugin-focused tests for the newer Hermes plugin architecture.
    - **macOS gateway sandbox support:** added sandbox profile generation, `hermes-aegis test-sandbox`, and patches for safer local gateway sessions while preserving the local gateway backend.
    - **Hybrid migration helpers:** added migration utilities and tests for moving existing installs toward the hybrid plugin layout.
    
    **Proxy/provider fixes**
    - Fixed OAuth/provider auth refresh propagation when starting the proxy through `hermes-aegis`, so Hermes-minted credentials can refresh correctly.
    - Refreshed provider allowlists for current Hermes endpoints, including Nous/Z.AI related hosts.
    - Added TCP passthrough / proxy-bypass fixes for local or streaming services that should not be MITM-wrapped.
    - Unblocked web tools behind the Aegis proxy by fixing TLS/allowlist coverage.
    
    **Security and audit improvements**
    - Docker operations are now logged as dangerous-command activity in the audit trail, making container-risk behavior visible to reports and reactive rules.
    - Updated Docker/network isolation tests so they do not collide with live Aegis networks.
    - Added more regression coverage around sandboxing, migration, provider injection, redaction, and update integration.
    
    **Docs/release cleanup**
    - Version metadata is now aligned at `0.2.0` across the package and plugin.
    - README compatibility guidance was refreshed from old Hermes v0.3-era wording to current Hermes v0.11 guidance.
    - Patch documentation now reflects the new update integration and current patch count.
    
    **Verified release state**
    - Full test suite: `919+ passed, 8 skipped` during release prep.
    - Release tag: `v0.2.0`
    
    **Update / run**
    ```bash
    hermes-aegis update
    hermes-aegis
    ```
    
    This release is mainly about making Aegis feel seamless again: update once, launch simply, keep the shield wrapped around Hermes, and stay compatible with the fast-moving upstream codebase.
    
    v0.3.0 is being planned out and will focus on improving the security further.
