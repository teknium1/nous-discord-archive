    **hermelinChat** - a browser UI for Hermes Agent that doesn't sacrifice the terminal
    
    Started this for the first Nous Research Hermes Agent Hackathon a while back, but it's grown quite a bit since then.
    
    **What it does:**
    • Real PTY-backed terminal via xterm.js - every keystroke goes to the actual `hermes` process. Tab completion, Ctrl-C, scrollback - all of it
    • Session sidebar that reads directly from `~/.hermes/state.db` - browse, search, and resume past sessions
    • Artifact panel - Hermes can create and manage HTML/iframe/code artifacts, with sandboxed runner proxying so complex artifacts work remotely too
    • Built-in Strudel editor (opt-in) with agent toolset controls
    • Single-port deployment - FastAPI serves the React SPA, WebSocket PTY proxy, and API all in one
    
    **Stack:** Python (FastAPI) + React + xterm.js + SQLite
    
    🔗 https://github.com/quarker1337/hermelinChat
    🔗 https://www.youtube.com/watch?v=wnbQeIG-AUo
    
    Happy to answer questions - and yes, large parts of this were built *with* Hermes-Agent itself. (And Hermes-Agent is still updating it.)
