    Hello everyone, I'm really glad to show GVM: a runtime governance layer for Hermes Agent — E2E test pass, 0 false blocks
    I've been running Hermes in ec2 and it works great. But there's always same anxiety, what if it does it shouldn't? 
    
    So I built a governance runtime that restricts what the agent can actually do. You define a policy (allowed endpoints, methods, payload patterns), and the agent cannot reach anything outside that policy. It's not a suggestion to the LLM — it's kernel-level enforcement that the agent can't override or bypass.
    
    Ran the same E2E test pipeline I used for Hermes passed cleanly.(on EC2)
    
    The agent runs inside a Linux sandbox (namespace + seccomp isolation), so it can't circumvent the governance layer even by spawning subprocesses or making raw HTTP calls. Single Rust binary, nothing to configure besides a TOML policy file.
    GitHub: https://github.com/skwuwu/Analemma-GVM
    Curious how others running Hermes long-term handle this — are you relying on tool allowlists alone, or do you have network-level enforcement? Welcome for feedbacks!
