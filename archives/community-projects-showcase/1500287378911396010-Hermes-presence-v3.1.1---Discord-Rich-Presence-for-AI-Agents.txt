    hermes-presence v3.1.1 is on PyPI. pip install hermes-presence
        
        Discord Rich Presence that tracks what your Hermes Agent is doing in real time — tool calls, model, provider, session cost, sub-agent count. Shows up as "Playing Hermes Agent" with live activity.
        
        Full feature set:
        
        - Real-time presence — tool name, model, provider, cost ($), sub-agent count, elapsed timer
        - Tool-specific icons — terminal → working, browser → monitoring, read_file → researching, memory → standby
        - Multi-pipe connections — stable + canary pipes, auto-reconnect
        - Multi-profile — separate presences per Hermes profile (--profile research) with isolated state files
        - Cross-platform — Linux (systemd), macOS (launchd), Windows (Scheduled Task + .bat), WSL2 bridge
        - WSL→Windows mirror — profile-aware state copied to %APPDATA% for native Windows apps
        - Session metrics — tool count, sub-agents, files modified, session cost, uptime
        - Webhook notifications — POST on state changes (idle → thinking → calling → done → error)
        - Self-update — hermes-presence update pulls latest from GitHub and hot-reloads
        - Rich CLI — status --json, restart, validate, doctor, config set, run --profile
        - Hermes hook integration — post_llm_call hook fixes the "unknown model" display bug
        - Configurable — idle timeout, poll interval, tool exclusion, custom buttons (GitHub, dashboard, etc.)
        - Error detection — catches crashes, shows error messages in Discord presence
        - Cron/orchestrator-aware — detected via env vars, displayed accordingly
        - Atomic writes — temp-file + rename for state integrity
        
        bash
        pip install hermes-presence
        hermes-presence install --client-id YOUR_DISCORD_CLIENT_ID
        hermes-presence status
