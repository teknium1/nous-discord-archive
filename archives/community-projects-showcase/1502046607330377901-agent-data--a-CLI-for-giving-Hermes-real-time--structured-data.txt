    agent-data is a CLI tool that gives Hermes access to real-time, structured data without relying on browser automation.
    
    **The goal** is to enable Hermes to handle tasks requiring live data like “find me a roundtrip flight from NY to SF next month” without wasting tokens parsing HTML or interpreting browser screenshots. 
    
    **The current version supports:**
    - Real-time data for categories like flights, restaurants, RSS/blog feeds, and social media feeds
    - Structured JSON outputs by default
    - Endpoint discovery so Hermes can find relevant capabilities at runtime instead of loading tool schemas into context at every turn
    - A CLI interface designed for agent workflows
    
    **We built this because alternatives fell short in a few ways:**
    - browser automation is generally slow, expensive and flaky
    - Ad-hoc scraping works well in narrow cases, but can become quite expensive at scale
    - MCP servers can inflate context when loading schemas on every turn, and are often either too granular or too coarse in their design
    
    It’s still early and some parts are a bit rough, but we’re adding new endpoints and expanding the scope of existing ones every week.
    
    **I’d love to know**: 
    - Are there any live data sources you wish Hermes could access (e.g., aggregated job postings, hotel listings, firmographics, etc.)? 
    - How has browser automation, scraping, generic web search, or MCP tools failed when using Hermes?
    
    **I’m also looking for early users to test agent-data with real Hermes workflows! Please DM or comment if you're interested!**
