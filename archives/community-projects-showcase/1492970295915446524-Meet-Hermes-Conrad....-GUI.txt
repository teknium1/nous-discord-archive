    I have not coded in 20 years, Claude code and hermes have renewed my interest in trying to make things. This is vibe coded.
    
    I built ConradGUI, a local-first dashboard for Hermes Agent that gives you a proper operator view of what the agent is actually doing. It combines the normal chat interface with real inspection surfaces for memory, skills, cron jobs, config, and full session history — all pulled directly from ~/.hermes/ via a read-only companion backend. Chat still runs through the standard Hermes API, but everything else comes from the actual on-disk state, so what you see is always grounded in reality.
    
    The goal was to avoid fake or duplicated logic and instead make Hermes more transparent. You can browse all historical sessions (including terminal/cron/API runs), watch memory and config update live, and respond to approval flows directly in chat with inline actions + notifications. It runs entirely on your own machine (Docker or local), no cloud, no accounts, no telemetry. Basically trying to make Hermes feel like a real system you can operate, not just a chat box.
