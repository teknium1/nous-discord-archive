    https://github.com/wysie/mnemosyne-dashboard
    
    Mnemosyne Dashboard — Features
    
    - Local, read-only web dashboard for browsing and visualising Mnemosyne memory stores
    - Opens the SQLite database in read-only mode; no edit/delete/write memory endpoints
    - No cloud calls, no external JS/CSS dependencies, self-hosted static assets
    
    Core views:
    - Overview dashboard with counts for working memory, episodic memory, triples, and consolidations
    - Recent memory cards with raw JSON detail drawer
    - Memory browser with query, tier, source, scope, session filters, and sorting
    - Global search across memories, triples, and consolidations
    - Triples table with clickable row details
    - Recall debugger with approximate ranking explanations
    - Timeline grouped by day or session
    - Consolidation history with filtering, JSON inspection, and jump-to-session memories
    
    Graph / exploration:
    - Interactive relationship graph
    - Query filter for graph data
    - Clickable nodes and edges
    - Inspector panel with jumps into Triples and Memories
    - Clickable overview breakdown rows that jump into filtered Memories
    
    UI / UX:
    - Desktop and mobile responsive layouts
    - Dark and light themes
    - Mobile top bar with hamburger menu and theme toggle
    - Mnemosyne-inspired light theme with self-hosted fonts/assets
    - Screenshot gallery in README generated from synthetic mock data
    
    Security / ops:
    - Binds to 127.0.0.1 by default
    - Can bind to 0.0.0.0 only when explicitly configured for LAN access
    - Optional password authentication from Settings
    - /api/health endpoint for smoke checks and uptime probes
    - Baseline browser security headers: CSP, no-sniff, frame-deny, no-referrer
    - Hardened static asset serving with path traversal protection
    
    Hermes plugin tools:
    - mnemosyne_dashboard_start
    - mnemosyne_dashboard_stop
    - mnemosyne_dashboard_status
    - mnemosyne_dashboard_config
