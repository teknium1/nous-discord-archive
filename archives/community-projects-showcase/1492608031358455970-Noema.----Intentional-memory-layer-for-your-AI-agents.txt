    Hey Folks,
    
    TL;DR: I'm sharing this today: https://noemacortex.com
    
    I started writing a little tool for myself to quickly record daily events — something lightweight I could use at work to keep track of small wins so I'm not blanking when review time comes around. Anybody else been there too?
    
    As AI agents became more of a daily thing in my world, I realized the same tool worked well to give agents persistent memory across sessions. That became the basis for Noema: structured knowledge stored as plain markdown files, indexed by SQLite, no cloud dependency. Everything lives on your machine.
    
    Where it got interesting: I run agents across multiple machines, and they kept rediscovering the same things independently. So I added federation — cortexes (Noema's term for a memory collection) can peer together and sync knowledge automatically.
    
    Publishers can broadcast read-only knowledge; subscribers consume it. A research agent's findings become available to a coding agent without anyone re-teaching it. **Agents teaching agents**.
    
    It's written in Go, ships as a single binary, and works with any MCP-compatible agent (Hermes, Claude Code, Copilot, Zed, etc.).
    
    `brew install Fail-Safe/noema/noema`
    
    GitHub: https://github.com/Fail-Safe/Noema
    
    Thanks for looking — happy to answer questions about the architecture or federation design. Cheers!
