    I built Agent Feeds for Hermes.
    
      If Agent Skills teach Hermes how to act, Agent Feeds help Hermes stay aware of what’s already happening around it.
    
      The output is simple: Hermes gets a compact list of available local streams, and detailed data stays in JSON state files under ~/.agentfeeds. When a question is
      relevant, Hermes reads the right local file instead of re-fetching context or searching the web.
    
      That means:
    
      - faster responses from already-refreshed local state
      - fewer tokens spent carrying bulky context every turn
      - easier onboarding for a new Hermes setup, because subscriptions and state live in one inspectable local directory
      - less repeated manual setup across conversations
      - a clean path for adding private/local sources through providers
    
      Current providers include local files, RSS/Atom, Hacker News, GitHub issues/PRs/releases, ICS calendars, weather, exchange rates, and safe argv-only local commands.
    
      Example Hermes UX:
      “Subscribe my project notes at ~/notes/project.md.”
      “Subscribe me to Hacker News front page.”
      “Refresh Project notes and summarize it.”
      “Can Agent Feeds subscribe to my SQLite task database? If not, draft a provider.”
    
      Repo: https://github.com/verkyyi/agentfeeds
    
      Curious what streams other Hermes operators would want their agents to stay aware of.
