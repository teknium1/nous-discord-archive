    Everything in MEMORY.md and USER.md gets shipped to your LLM provider on every single turn. If you ever said "remember my Postgres password" to Hermes, OpenAI / OpenRouter / Nous Portal have it now.
    
    VEIL is a Hermes skill that intercepts sensitive values before they reach memory, encrypts them locally (XChaCha20-Poly1305 + Argon2id), and resolves them back to plaintext only inside child processes the agent spawns. The LLM never sees the real value - only a {{veil:ref}} placeholder.
    
    Optional: mirror vault integrity to Solana via the Memo program for tamper-evidence. ~$0.001 per publish.
    
    What I tested:
    - 34 / 34 pytest unit tests
    - 18 / 18 smoke tests on a clean Ubuntu VPS (Docker)
    - 4 / 4 LLM trigger tests against claude-sonnet-4.6, including the critical negative test (non-sensitive data must NOT be sealed - it wasn't, and the agent explained itself: "Saved in plain memory - no sensitive data there.")
    
    Install:
    hermes skills install github:Zhekinmaksim/hermes-veil
    hermes /veil init
    
    Repo, threat model, demo script: https://github.com/Zhekinmaksim/hermes-veil
    
    Screenshots:
    1. Explicit privacy cue -> OpenAI key sealed into a {{veil:ref}} placeholder
    2. Pattern-only trigger -> GitHub PAT sealed even without an explicit privacy cue
    3. PII + financial context -> client data sealed as pii
    4. Negative test -> benign preference stays in plain memory, no VEIL trigger
    5. Final vault state -> only sensitive entries are stored in the vault
    
    MIT. Feedback welcome - especially on patterns missing from the detector.
