    We just pushed a big update to the RetainDB integration.
    
    This goes beyond “memory that works” and turns Hermes into a system where agents can persist and share context across sessions and across each other.
    
    What’s new
    
    Reliable memory
    All routes fixed and writes are now durable. Every turn is queued locally before sending, so nothing is lost on crashes or network issues.
    
    No latency
    Memory writes don’t block the agent loop anymore. Everything flushes in the background with retry.
    
    Smarter context
    Instead of just recall, RetainDB prepares context ahead of time:
    
    relevant long term memory
    a synthesized understanding of the user
    a persistent self model for the agent
    
    Shared files
    Agents now share a file system:
    rdb://files/org/...
    
    They can upload, read, list, ingest, and delete files. Anything uploaded by one agent is available to others.
    
    Files become memory
    After ingest, documents are parsed, embedded, and turned into memory. They show up in the same memory search. No separate pipeline.
    
    What this enables
    
    agents that don’t reset every session
    real multi agent workflows
    documents that become usable context
    shared knowledge across a project or org
    
    Most tools stop at storing chats.
