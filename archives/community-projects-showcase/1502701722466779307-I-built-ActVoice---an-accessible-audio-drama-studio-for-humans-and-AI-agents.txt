    I built ActVoice — an accessible audio drama studio for humans and AI agents.
    
    Website: https://actvoice.xyz
    Source: https://github.com/web3blind/actvoice
    
    It turns structured text into a rendered audio performance. Instead of editing a visual timeline, you describe the project as data: characters, voices, scenes, dialogue, ambience, sound cues, timing anchors, render jobs, and MP3/WAV artifacts.
    
    Why I built it:
    I’m a blind developer, and many creative tools assume creation means seeing and manipulating a timeline. I wanted the opposite: a workflow driven through text, a screen reader, REST API, MCP, or an AI agent.
    
    For agents, this is a natural fit: plan a scene, assign voices, place sound cues, render, inspect the manifest, and iterate without a visual editor.
    
    What works already:
    - bilingual homepage with browser language auto-detection
    - copy-ready commands
    - REST API + local MCP workflow
    - agent registration + bearer keys
    - project/scene/character/dialogue APIs
    - render queue with polling
    - downloadable MP3/WAV artifacts + manifest
    - Microsoft Edge TTS voices + RHVoice fallback
    - Openverse ambience/SFX search
    - semantic cues like brook, footsteps, birds
    - timing via start_ms or anchors like after_line + line_id
    
    Important design choice: ActVoice has no built-in “core AI”.
    
    The service stays deterministic: store project data, render voices, resolve timing, fetch/mix ambience and SFX, and export artifacts.
    
    The intelligence comes from outside: Hermes Agent, MCP clients, other agents, or human-authored manifests. That makes it easier to test, reason about, and adapt to different agent stacks.
    
    I already used ActVoice to create a personal audio drama. The attached English sample is a rendered example converted to MP4 for sharing.
    
    Built with Hermes Agent by Nous Research.
    
    Created after developing the hermes-audiodrama skill, which I previously wrote about in my hackathon thread.
