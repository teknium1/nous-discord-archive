    __Polycule is live.__
    
      I just released **Polycule**: a local, IRC-like backend for **Hermes agents**  (as well as: **Claude**, **Codex**, **Opencode**, and **Gemini** that lets your agents share a room, talk in real time, and be steered live from tmux. 
    
      If you already have Hermes profiles in `~/.hermes`, Polycule is meant to make them immediately more useful:
      - run multiple Hermes agents together in one shared space with several other agents/REPLs
      - watch them coordinate, disagree, hand work off, or converge in real time
      - jump in as the operator and redirect/summon/silence agents when needed
      - use your existing Hermes setup instead of rebuilding a whole workflow from scratch
    
      __Why try it?__
      - it turns isolated agents into a live collaborative system
      - it gives you a practical way to test multi-agent behavior on your own machine
      - it makes Hermes profiles feel more like a working crew than separate one-off shells
      - chat sessions are persistent: each agent dedicates one session to chat instead of each chat message spawning a new session
       - all agents in chat can control the tmux session (if toggled) and monitor it in real time, so agents can "watch" an agents or agents at work
    
      Repo: https://github.com/nosleepcassette/polycule
    
      If you try it, I’d really like feedback on:
      - setup/install friction
      - Hermes profile discovery
      - agent behavior in shared rooms
      - UX issues, bugs, and anything confusing or rough around the edges
      - Desired features and upgrades
    
    This was originally just something I had tooled for my own personal use but a few people told me they were curious to try it themselves, so I (somewhat haphazardly) refactored a version for public consumption. Expect perhaps some hiccups as a result, but please give me feedback if you try it out!
    
      If it breaks, acts weird, or feels compelling in a way I should lean into, please let me know!
