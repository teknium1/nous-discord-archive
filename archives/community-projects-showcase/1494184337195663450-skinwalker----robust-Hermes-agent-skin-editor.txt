    # skinwalker — Hermes skin editor, now with working spinner faces
    
      Built a full TUI skin editor for Hermes. Covers the entire skin surface: identity, palette, spinner, logo/hero art, AI-assisted generation. Ships today.
    
      **what it does**
      - browse, edit, and activate built-in and custom skins
      - live terminal-native preview + YAML preview side by side
      - palette presets, direct color editing, import/export
      - logo generation via pyfiglet with categorized font browser
      - image-to-ASCII hero generation via Pillow
      - spinner preset shelf + wing pair editor
      - emoji/symbol picker for any text field
      - AI-assisted branding, spinner, and art generation (via Hermes or any API backend)
      - undo/redo across the whole session
    
      **the bug it also fixes**
      Hermes has a bug where `waiting_faces`, `thinking_faces`, and `thinking_verbs` in your skin YAML are loaded but never applied — every session uses hardcoded kawaii faces regardless of your skin config. The repo includes a patch script
       that fixes it locally while the upstream PR is under review.
    
      ```zsh
      zsh patch-hermes.zsh
    
      patches display.py, run_agent.py, and cli.py. idempotent, backs up originals.
    
      install
      git clone https://github.com/nosleepcassette/skinwalker
      cd skinwalker
      zsh install.zsh --patch-hermes
    
      requires uv. drops skinwalker onto your PATH as a standalone command.
    
      upstream fix tracked at NousResearch/hermes-agent#10668

    • Release update: imagewalker + Skinwalker art upgrades
    
      Shipped a pretty large art/workflow pass today:
    
      - Added imagewalker, a standalone local image-to-ASCII app
      - Put it on-path locally as imagewalker
      - Added Image Lab integration inside Skinwalker’s Art tab
      - Added Open Imagewalker from Skinwalker for direct handoff into the standalone tool
      - Expanded hero/image controls with:
          - dithering
          - saturation / hue / grayscale / sepia
          - space density
          - better export flow
      - Added TXT export for generated logos and heroes
      - Added PNG export for generated logos and heroes
      - Fixed logo preview workflow so previews render live in the main Preview pane instead of getting mangled inside the narrow input/editor area
      - Improved logo font browsing with more explicit, human-readable categories
      - Fixed preview-time left / center / right alignment so logos/heroes line up against the actual UI borders instead of just baking weird whitespace into the art
    
      Standalone imagewalker currently supports:
    
      - Textual TUI
      - CLI print mode
      - 20-400 character widths
      - multiple gradients
      - dithering modes
      - TXT / PNG export
      - clipboard copy
    
      Known limitation
    
      - Truly responsive heroes for different tmux pane widths are not solved yet at the Hermes runtime level.
      - Skinwalker can generate better static art now, but real per-pane reflow likely needs a Hermes-side runtime change.
