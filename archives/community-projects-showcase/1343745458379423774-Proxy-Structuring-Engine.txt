    I built the Proxy Structuring Engine (PSE) because I wanted better structured outputs from my LLMs, and I didn't like the existing libraries.
    
    So, I made my own.
    
    The PSE guarantees 100% structurally valid outputs (JSON, XML, or any format) during generation, using inference-time steering. It's like guardrails for your model, guiding their output without limiting their creativity.
    
    Our benchmarks show the PSE outperforms existing solutions like Outlines and LM-Format-Enforcer, delivering higher quality generations and faster generation times.
    
    The PSE is open source (Apache 2.0) and it should be easy plug and plug the PSE with your local models.
    
    * Install: `pip install pse`
    
    More details: https://proxy.ing/pse
    
    
    GitHub: https://github.com/TheProxyCompany/proxy-structuring-engine
    
    Announcement Thread: https://x.com/TheProxyCompany/status/1894123631531135338
    
    I'm happy to answer any questions about the implementation, use cases, or how it compares to other approaches. Let me know what you think!

[2025-02-27T13:18:20.511000+00:00] jckwind (id=1344659890681151560)
    a couple high level points - first the way I’m building the structure. I’m using a custom hierarchical state machine instead of regex or grammars (although those individual features are supported) - so it can handle much more complex schema & structure. The full json specification (I think the PSE like 80% there, implementing keywords like ‘not’ would be more effort than it’d be worth)
    
    The PSE also is calculating the logit mask on the fly instead of building a map of all valid tokens for a given schema. This gives us a big boost in speed.

[2025-02-27T13:23:27.098000+00:00] jckwind (id=1344661176600105062)
    Another thing it does is constrain random sampling - what this means is the PSE wraps any given sampler method (a callable that takes logprobs and returns a token_id) and will validate this token through the HSM. There’s some resample logic, and we are exploring multiple paths through the schema at once. 
    
    I won’t go into deep specifics (I’ve spoken about the architecture other places) but it’s not doing backtracking. Main ideology I had while building the PSE was maximizing the influence the underlying probability distribution had while still structuring the output.
